# Brain Layer Add-on: GTM

Use this file to reason about market entry and expansion.

## 1. GTM definition

GTM is the path by which the first customers understand, trust, try, buy, and expand usage of the product.

## 2. Core principles

### Principle A: Start with the first painful wedge
Define a small, painful, clearly reachable starting segment.

### Principle B: Trust is part of GTM
Especially in B2B, customers buy after they trust the supplier, the workflow, and the risk profile.

### Principle C: Activation matters as much as acquisition
A user who arrives but never reaches value is not real progress.

### Principle D: Expansion is half the GTM story
Describe what happens after the first sale: more users, more teams, more modules, more workflows.

## 3. Required questions

1. Who is the first customer wedge?
2. Why will they trust us?
3. What is the first value moment?
4. What is the most suitable channel at this stage?
5. How do we expand after the first sale?
6. What is the weakest GTM assumption?

## 4. Anti-patterns

- listing every channel
- writing only about acquisition
- assuming AI demos create trust automatically
- no expansion path inside the account
