# Hands Layer: Product Plan Workflow

Use this file to run the internal multi-analyst process.

## Optional pre-step: Batch screening mode

If the user provides multiple candidate ideas, do not draft full product plans for all of them immediately.
Instead:
1. create a light commercial brief for each idea
2. score each on market tension, value logic, commercial model, GTM realism, competition/defensibility, risk honesty, and evidence potential
3. rank them comparatively
4. shortlist the top 3-5 ideas
5. only then draft full product plans for the finalists

## Step 0: Build the commercial brief

Create this structure first:

```md
# Commercial Brief
- Product thesis:
- Target market:
- Buyer and user:
- Current alternatives:
- Business stage:
- Constraints:
- Biggest unknowns:
- Assumptions:
```

## Step 1: Parallel analyst pass

Run five internal analysts.

### Market Analyst
- why does this market move now?
- where is the real tension?

### Business Model Analyst
- how does value become revenue?
- what must be true for the economics to work?

### GTM Analyst
- how does the first wedge get acquired and activated?
- how does the account expand?

### Competition Analyst
- what substitutes exist?
- why has this not already been won by incumbents?

### AI Commercialization Analyst
Use only if AI is core.
- does AI improve value capture or only product perception?
- where are the cost and trust risks?

## Step 2: Cross-examination

Each analyst must challenge another lens.

## Step 3: Conflict table

Summarize into:
- agreements
- key disagreements
- overclaimed narratives to avoid
- top three risks

## Step 4: Convergence

Create:
- one recommended business path
- one fallback path
- the logic for choosing the winner

## Step 5: Hand off to Output layer

Only then draft the final product plan.
