# Hands Layer: Positioning Workflow

Use this file to run the internal multi-analyst process.

## Optional pre-step: Batch screening mode

If the user gives multiple product ideas, do not write full positioning documents for all of them first.
Instead:
1. create a short brief for each idea
2. score each idea on ICP clarity, problem sharpness, alternative clarity, category frame, differentiation, why-now strength, and strategic coherence
3. rank them
4. shortlist the top 3-5 candidates
5. only then run the full workflow on the shortlisted ideas

## Step 0: Standardize the brief

Create this structure first:

```md
# Positioning Brief
- Product idea:
- Target user:
- Core scenario:
- Stage:
- Constraints:
- Biggest unknown:
- Assumptions:
```

## Step 1: Parallel analyst pass

Run four internal analysts.

### PMF Analyst
Answer:
- is the pain strong enough?
- why would users switch?
- what wedge is most plausible?

### Positioning Analyst
Answer:
- what category frame fits?
- what is the differentiated value?
- what wording is too broad or weak?

### AI Product Analyst
Use only if AI is core.
Answer:
- where is AI embedded?
- what trust boundary exists?
- does AI actually change the workflow?

### Narrative Analyst
Answer:
- can this be explained in one sentence?
- does the story guide product decisions?
- what part of the narrative is still fuzzy?

## Step 2: Cross-examination

Each analyst must challenge at least one other analyst:
- what is missing
- what is too optimistic
- what is too broad
- what would break in the market

## Step 3: Conflict table

Summarize into:
- agreements
- main disagreements
- rejected directions
- key assumptions kept alive

## Step 4: Convergence

Create:
- one recommended positioning
- one fallback positioning
- a short reason for choosing the winner

## Step 5: Hand off to Output layer

Only after convergence should you draft the final document.
