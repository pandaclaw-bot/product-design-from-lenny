## 一句话介绍

我们这次把“产品想法 → 产品定位 → 商业计划书”这条链路，拆成了两个正式 Skill：

- **lenny-product-positioning**：负责把想法收敛成清晰定位
- **lenny-business-plan**：负责把定位扩展成商业计划书

它们不是直接生成答案，而是会经过：
**方法论判断 → 多智能体讨论 → 文档生成 → 评估打分 → 不过线回炉**。

---

# 示意图 1：两个 Skill 分工图

```text
[产品想法]
    ↓
[lenny-product-positioning]
    ↓ 输出：产品定位说明
[lenny-business-plan]
    ↓ 输出：商业计划书
[最终结果]
```

## 配文
第一个 Skill 负责把“这个产品到底做什么、给谁做、为什么成立”想清楚；
第二个 Skill 负责把“这是不是一门生意、怎么卖、谁付钱、风险在哪”讲清楚。

# 示意图 2：每个 Skill 的内部结构

```text
┌───────────────┐
│   Brain       │  方法论 / 判断框架 / rubric
├───────────────┤
│   Hands       │  多智能体讨论 / 辩论 / 收敛
├───────────────┤
│   Output      │  文档生成 / 专业表达 / 审核
├───────────────┤
│ Evaluation    │  打分 / 过线判断 / 回炉重构
└───────────────┘
```

## 配文
这套 Skill 的核心不是“写得快”，而是“写之前先想清楚，写完后再自评，不达标就继续改”。

---

# 示意图 3：正确使用顺序

```text
单个想法：
产品想法
  → 产品定位 Skill
  → 通过 evaluation
  → 商业计划书 Skill
  → 通过 evaluation
  → 最终交付
```

## 配文
先定位，再写商业计划书。不能反过来。
如果定位没收清，商业计划书大概率会变成一份看起来很完整、实际很空的作文。

---

# 示意图 4：多个想法时怎么用

```text
多个想法
  → Batch Screening（轻量筛选）
  → Top 3 / Top 5
  → 对入围方向做完整定位
  → 对入围方向做完整商业计划书
```

## 配文
这两个 Skill 现在支持“批量筛选模式”。
当你一次给 5 个、10 个方向时，不是每个都写长文档，而是先横向比较，再挑最值得深挖的方向。

---

# 示意图 5：Evaluation 机制

```text
生成初稿
   ↓
Evaluation 打分
   ↓
是否达到阈值？
   ├─ 是 → 交付最终版
   └─ 否 → 回炉重构 → 再评估
```

## 配文
这套 Skill 的关键不是“出稿”，而是“不过线不交稿”。

默认阈值：
- 产品定位：7/10
- 商业计划书（内部战略版）：7/10
- 商业计划书（对外包装版）：8/10

## 一段可直接发群的短文案

我们最近把“产品方向判断”这件事，收成了两个正式 Skill：

1. **lenny-product-positioning**：把一个模糊想法收敛成清晰定位
2. **lenny-business-plan**：把定位扩展成商业计划书

每个 Skill 内部都不是直接写答案，而是分成 4 层：
- Brain：方法论判断
- Hands：多智能体讨论与收敛
- Output：文档生成
- Evaluation：打分，不达标就回炉

也就是说，这套东西更像一个“会分析、会辩论、会写、还会自我把关”的策略顾问，而不是普通文档生成器。

如果一次有很多产品想法，它还可以先做 batch screening，筛出 Top 3 / Top 5，再做完整深挖。

---

## 最后一行总结

> 先想清楚做什么，再想清楚怎么成为一门生意；如果分数不过线，就继续回炉，直到拿到通过版。
