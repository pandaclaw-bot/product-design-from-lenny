# 三段式 Skill 架构蓝图 v2

- 日期：2026-03-13
- 状态：新版正式蓝图
- 目的：在“产品定位 skill + 商业计划书 skill”基础上，进一步引入三段式结构：**大脑 / 手 / 输出审核**

---

## 1. 为什么要升级成三段式

前一版蓝图已经把 skill 收缩成两个正式技能：
- `lenny-product-positioning`
- `lenny-business-plan`

但还不够。

你提出的约束非常关键：

> skill 不能只有“摘要”和“输出模板”，还需要把：
> 1. 分析提炼后的理念与方法论
> 2. 具体执行方法
> 3. 最终输出物的生成与审核技巧
> 三者拆开。

这是对的。因为真正可复用的 skill，不只是“知道一些观点”，而是必须同时拥有：

### 大脑（Brain）
知道怎么思考、如何判断、用什么框架做分析。

### 手（Hands）
知道怎么推进任务、怎么组织多智能体讨论、怎么收敛。

### 输出审核（Writer + Reviewer）
知道怎么把内容写成专业、标准、可交付的文档，并进行最后的润色与审校。

---

## 2. 新版总结构

新版仍然只有 **2 个正式用户侧 skill**，但每个 skill 内部都包含 3 个子层：

### Skill A
`lenny-product-positioning`

### Skill B
`lenny-business-plan`

### 每个 skill 的内部结构
1. **Brain layer**：理念、方法论、判断标准
2. **Hands layer**：执行方法、多智能体辩论与收敛流程
3. **Output layer**：文档生成技巧、审核清单、润色标准

这意味着：
- 对外只有两个 skill，入口清晰
- 对内每个 skill 都不是单线程，而是三段式协作体系

---

## 3. 三段式的定义

# 3.1 Brain layer（大脑）

## 职责
把原始访谈和派生摘要进一步提炼成：
- 可复用的理念
- 判断框架
- 分析维度
- 方法论检查表

## 它不做什么
- 不直接输出最终文档
- 不负责长篇写作
- 不负责执行细节讨论

## 典型内容
- positioning 方法论
- PMF 判断原则
- GTM 第一性原理
- 定价逻辑
- 企业 AI 产品的信任边界原则

## 在 skill 中的表现形式
主要放在：
- `references/*.md`
- 方法论清单
- 核心观点矩阵
- 评价 rubric

---

# 3.2 Hands layer（手）

## 职责
把“知道怎么想”变成“知道怎么做”。

它应该负责：
- 接收用户输入
- 标准化 brief
- 组织多智能体讨论
- 制造分歧
- 推动反驳
- 形成一致结论

## 这里最关键的要求
你要求这里要有：
- 多智能体互相辩论
- 互相讨论
- 形成一致意见
- 有讨论、有审核、最后达成一致

这个要求我完全认同。

所以 Hands layer 不应该只是 workflow list，而应该是：

### 多智能体执行协议（Multi-agent Execution Protocol）

包含：
1. **Brief 标准化**
2. **并行分析**
3. **交叉质询**
4. **冲突整理**
5. **共识收敛**
6. **交给输出层**

## Hands layer 的本质
它不是“怎么写文档”，而是“怎么把复杂判断问题推进到可以写文档”。

---

# 3.3 Output layer（输出生成与审核）

## 职责
把已经相对成熟的共识，变成一个：
- 结构清晰
- 专业标准
- 语言稳定
- 风格一致
- 可交付的输出物

## 它包含两类能力
### A. 文档生成技巧
- 怎么组织结构
- 怎么写 executive summary
- 怎么避免空话
- 怎么保持专业感
- 怎么让内容更适合被阅读、被汇报、被讨论

### B. 审核与润色技巧
- 逻辑是否闭环
- 是否有前后冲突
- 是否有 unsupported claims
- 是否有模板腔
- 是否缺少行业语言/专业表达
- 是否需要进一步压缩或增强说服力

## 结论
Output layer 不只是“排版”，而是：

> **最后一道专业输出质检层**

## 4. 共享派生参考层的新角色

前一版我们已经有：
- `01-核心观点分类摘要-v1.md`
- `02-skill-reference-拆分建议.md`
- `03-新skill蓝图-v1.md`

现在开始，这些材料主要服务于 **Brain layer**。

也就是说：
- transcript 原文 → 不直接喂给 skill
- 分类摘要 → 进入 Brain layer 的原料
- Brain 再整理出 skill 专用方法论参考

---

## 5. Skill A：lenny-product-positioning（新版）

# 5.1 目标
把一个模糊的产品想法，收敛成：
- 明确定位
- ICP
- 核心价值主张
- category frame
- 差异化
- 切入场景
- 战略路径

---

# 5.2 Brain layer

## 需要的核心参考
建议整理成以下参考文件：
- `references/pmf-core-views.md`
- `references/positioning-core-views.md`
- `references/ai-product-core-views.md`（如果 AI 是核心）
- `references/product-narrative-core-views.md`

## 核心方法论模块
1. **PMF 判断框架**
   - 痛点强度
   - 替代方案
   - 切换理由
   - 初始切口

2. **Positioning 框架**
   - 参照类别
   - 差异化价值
   - 为什么是你
   - 为什么是现在

3. **AI 产品嵌入框架**
   - AI 在哪一环最值钱
   - 哪些环节该自动、哪些该辅助
   - 信任边界

4. **叙事一致性框架**
   - 用户能否听懂
   - 团队能否围绕它协同
   - 市场是否能快速理解

---

# 5.3 Hands layer

## 多智能体执行方法
建议每次内部固定 4 个分析代理：

1. **PMF Analyst**
2. **Positioning Analyst**
3. **AI Product Analyst**（如果适用）
4. **Narrative Analyst**

## 执行流程
### Step 0：标准化输入
形成统一 brief：
- 产品想法
- 目标用户
- 核心场景
- 当前阶段
- 关键约束

### Step 1：并行初判
每个 analyst 独立输出：
- 核心判断
- 最大风险
- 最想收窄的点

### Step 2：交叉质询
每个 analyst 至少反驳另一个 analyst 一次：
- 你忽略了什么
- 为什么你的判断过宽/过浅/过乐观
- 如果按你说的做，会掉进什么坑

### Step 3：冲突整理
形成：
- 共识
- 核心分歧
- 被否掉的方向
- 保留的关键假设

### Step 4：收敛会议
由一个 **Synthesis Agent** 负责：
- 收敛为一个定位建议
- 给出一个备选定位
- 明确为什么主定位优于备选

### Step 5：交给 Output layer
把收敛结果交给写作与审核层。

---

# 5.4 Output layer

## 文档生成目标
输出一个标准化的《产品定位说明》。

## 固定输出结构
1. 一句话定位
2. ICP
3. 核心问题与替代方案
4. 核心价值主张
5. category frame
6. 差异化表达
7. 为什么现在成立
8. 战略路径

## 审核清单
输出前必须检查：
- 有没有一句话就能讲清
- 有没有把“用户是谁”写得太宽
- 有没有把功能当定位
- 有没有把 AI 当噱头而不是价值环节
- 有没有“听起来都对但记不住”的空话

## 风格要求
- 结构标准
- 语言专业
- 尽量短、准、可复述
- 适合作为后续商业计划书输入

## 6. Skill B：lenny-business-plan（新版）

# 6.1 目标
把一个已经初步明确定位的产品，扩展成一份商业计划书。

---

# 6.2 Brain layer

## 需要的核心参考
建议整理成以下参考文件：
- `references/business-plan-core-views.md`
- `references/gtm-core-views.md`
- `references/pricing-core-views.md`
- `references/competition-core-views.md`
- `references/ai-commercialization-core-views.md`

## 核心方法论模块
1. **市场机会框架**
   - 问题是否真实且普遍
   - 市场变化是否支持机会
   - 时间窗口是否存在

2. **商业模式框架**
   - 谁付钱
   - 为什么付
   - 价值如何与价格对齐
   - 毛利结构是否健康

3. **GTM 框架**
   - 第一批客户怎么来
   - 为什么会信任
   - self-serve vs sales-assisted
   - 组织内扩散路径

4. **竞争与壁垒框架**
   - 替代方案是谁
   - 为什么现在没有人把这件事做好
   - 结构性优势在哪

5. **AI 商业化框架**
   - AI 是否提高交付效率
   - AI 是否提升价值密度
   - AI 成本与质量是否可控

---

# 6.3 Hands layer

## 多智能体执行方法
建议每次内部固定 5 个分析代理：

1. **Market Analyst**
2. **Business Model Analyst**
3. **GTM Analyst**
4. **Competition Analyst**
5. **AI Commercialization Analyst**（如适用）

## 执行流程
### Step 0：接收定位输入
默认输入来自 `lenny-product-positioning` 的产物。
如果没有，则先做轻量定位收敛。

### Step 1：并行判断
每个 analyst 输出：
- 这个商业故事最强的一点
- 最大脆弱点
- 最需要验证的假设

### Step 2：交叉质询
强制互相挑战：
- 市场大不代表能卖
- 能卖不代表利润健康
- 有 AI 不代表护城河成立
- 能起步不代表能规模化

### Step 3：矛盾整理
输出：
- 当前商业逻辑成立的部分
- 最危险的 3 个假设
- 哪些故事不能讲过头

### Step 4：共识收敛
由 **Business Synthesis Agent** 输出：
- 商业主线
- 备选商业主线
- 为什么主线更优

### Step 5：交给 Output layer
交给写作与审核层生成正式商业计划书。

---

# 6.4 Output layer

## 文档生成目标
输出一份结构专业、叙事清晰的《商业计划书》。

## 固定输出结构
1. 执行摘要
2. 市场痛点与机会
3. 产品方案概述
4. 目标市场与客户画像
5. 商业模式与定价思路
6. GTM 策略
7. 竞争与替代分析
8. 风险与应对
9. 里程碑规划
10. 当前结论

## 审核清单
输出前必须检查：
- 市场机会是不是拍脑袋
- 商业模式是不是只写“订阅制”就结束
- GTM 是否过于空泛
- 风险部分是否只是形式化罗列
- 是否把“AI”当成壁垒本身
- 是否存在前后逻辑冲突

## 风格要求
- 更像专业商业文档，不像 brainstorming
- 能给团队、合作方、潜在投资人阅读
- 语言稳、少空话、强调逻辑链

---

## 7. 关于“文档生成技巧”是否要单独做 skill

当前建议：

### 先不把“文档生成”独立成第三个对外 skill
原因：
- 产品定位和商业计划书本来就各自需要自己的输出风格
- 如果再单独拆一个“通用润色 skill”，会让工作流反而更碎

### 更合理的做法
把“文档生成 + 审核技巧”做成每个 skill 内部的 Output layer。

也就是说：
- 不是第三个独立用户技能
- 而是每个正式 skill 内部都自带“写作与审核代理”

这样更清晰。

---

## 8. 最终形态

最终不是：
- 一堆平铺的 persona skill
- 或一个只会检索 transcript 的大 skill

而是：

### 两个正式 skill
1. `lenny-product-positioning`
2. `lenny-business-plan`

### 每个 skill 内部都有 3 层
- Brain：方法论与判断框架
- Hands：多智能体执行与辩论收敛
- Output：文档生成、审核、润色

### 共享输入材料层
- transcript 原文
- 分类摘要
- skill 专用方法论文档

这个结构比上一版更稳，也更接近你说的“像一个真正会思考、会执行、会出专业文档的 skill”。
