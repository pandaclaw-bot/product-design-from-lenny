# 两份正式 Skill 使用说明书 v1

- 日期：2026-03-14
- 适用对象：想用这套能力做产品方向判断、产品定位和商业计划书的人
- 当前正式 Skill：
  - `lenny-product-positioning`
  - `lenny-business-plan`

---

# 1. 先说人话：这两个 Skill 是干什么的？

你可以把它们理解成两个不同阶段的顾问。

## Skill 1：`lenny-product-positioning`
它负责回答：
- 这个产品到底做什么？
- 是给谁做的？
- 最适合先切哪个场景？
- 和现有替代方案比，有什么真正不同？
- 一句话应该怎么讲清楚？

一句话理解：
> **它负责把一个模糊想法，收敛成“明确的产品定位”。**

---

## Skill 2：`lenny-business-plan`
它负责回答：
- 这个产品是不是一门生意？
- 谁会付钱？
- 怎么收费更合理？
- 第一批客户从哪来？
- 为什么能做成，而不是停留在想法层？

一句话理解：
> **它负责把一个已经比较清楚的产品方向，扩展成“商业计划书”。**

---

# 2. 这两个 Skill 的正确使用顺序

## 正确顺序
### 第一步
先用：`lenny-product-positioning`

### 第二步
定位通过后，再用：`lenny-business-plan`

### 为什么不能反过来？
因为如果产品定位都没收清，直接写商业计划书，最后通常会变成：
- 看起来很完整
- 实际很空
- 市场很大、用户很多、路线很美
- 但没有真正锋利的切口

所以顺序一定是：

> **先想清楚做什么，再想清楚怎么成为一门生意。**

---

# 3. 这两个 Skill 不负责什么？

这点很重要，不然很容易误用。

## 它们不负责：
- PRD
- 页面原型
- 字段设计
- 数据模型
- 技术实现
- 详细交付方案
- 项目排期

也就是说：

> **这两个 Skill 只负责战略层，不负责执行层。**

如果你还在做“这个按钮放左边还是右边”，那已经不是这两个 Skill 的活了。

---

# 4. 它们内部是怎么工作的？

虽然对外你只看到两个 Skill，
但每个 Skill 内部其实都有 4 层：

## 1）Brain（大脑）
负责：
- 理念
- 方法论
- 判断框架
- rubric

## 2）Hands（手）
负责：
- 多智能体分析
- 互相质疑
- 收敛意见
- 形成判断

## 3）Output（输出）
负责：
- 把结果写成专业文档
- 结构清晰
- 表达专业
- 格式统一

## 4）Evaluation（评估）
负责：
- 打分
- 判断是否过线
- 不过线就回炉重构
- 只交付通过 evaluation 的版本

你可以把它理解成：

> 不是“随便写一版给你”，而是“先分析、再讨论、再写、再自评，过线了才交付”。

---

# 5. `lenny-product-positioning` 怎么用？

## 适合什么时候用
当你有这些问题时：
- 我这个产品到底怎么定位？
- 我该先切哪个客户？
- 一句话怎么讲清？
- 这个方向为什么现在成立？
- 它和现有方案到底有什么不同？

## 你给它的输入，最好长这样

```md
请帮我做产品定位：

- 产品想法：
- 目标用户：
- 核心场景：
- 当前阶段：
- 已知约束：
- 我最想想清楚的问题：
```

## 它会输出什么
通常会输出：
1. 一句话定位
2. ICP（目标客户）
3. 核心问题与替代方案
4. 核心价值主张
5. 类别参照（category frame）
6. 差异化表达
7. 为什么现在成立
8. 战略路径
9. 被否掉的定位方向
10. 保留假设与待验证问题
11. evaluation summary

## 你怎么判断它有没有帮到你
如果输出后你能清楚回答下面 3 个问题，就说明这个 Skill 用对了：
1. 这产品是给谁的？
2. 这产品到底解决什么问题？
3. 为什么不是另一个泛泛的软件工具？

---

# 6. `lenny-business-plan` 怎么用？

## 适合什么时候用
当你已经大概知道产品方向，想进一步回答：
- 这是不是一门生意？
- 谁会付钱？
- 怎么进入市场？
- 定价怎么想？
- 风险有哪些？

## 最好怎么输入
最理想的输入，是先把上一个 Skill 的输出作为输入。

例如：

```md
请基于下面这份产品定位，生成商业计划书：

- 一句话定位：
- 目标客户：
- 核心问题：
- 核心价值：
- 当前阶段：
- 已知约束：
```

## 它会输出什么
通常会输出：
1. 执行摘要
2. 市场痛点与机会
3. 产品方案概述
4. 目标市场与客户画像
5. 价值逻辑
6. 商业模式与定价逻辑
7. GTM 策略
8. 竞争与替代分析
9. 风险与应对
10. 里程碑规划
11. 当前结论（主路径 / 备选路径）
12. evaluation summary

## 你怎么判断它有没有帮到你
如果输出后你能清楚回答下面 4 个问题，就说明这个 Skill 用对了：
1. 谁会真的买单？
2. 为什么愿意买？
3. 第一批客户怎么拿？
4. 最大风险是什么？

---

# 7. Evaluation 是怎么回事？

这是这套 Skill 非常重要的地方。

## 它不是写完就交
每次输出前，Skill 都会先做一轮自评：
- 分数够不够
- 哪些地方太弱
- 要不要回炉重写

## 默认阈值
### `lenny-product-positioning`
默认：**7/10**

### `lenny-business-plan`
- 内部战略版：**7/10**
- 对外包装 / 投资人版：**8/10**

## 你也可以自己设阈值
例如：

```md
请用 business-plan skill 跑这件事，evaluation threshold = 7.5/10，低于分数就回炉，最后只给我通过版。
```

## 它交付时会附一个简短评估摘要
通常会告诉你：
- final score
- threshold used
- weakest remaining area
- suitability（更适合内部战略还是对外包装）

# 8. 当你一次有很多想法时，怎么用？

这两个 Skill 现在支持 **批量筛选模式（Batch Screening / Portfolio Mode）**。

## 正确做法
如果你一次有 5 个、10 个想法，不要让它每个都写长文档。正确方式是：

### 第一步
先让它做轻量筛选：
- 每个想法轻量打分
- 做横向比较
- 选出 Top 3 / Top 5

### 第二步
再对入围方向做完整深挖：
- 先定位
- 再商业计划书

## 示例
```md
我有 10 个 AI 产品想法，请先用 batch screening mode 做轻量定位和商业评估，筛出 Top 5，再只对前 2 名做完整结果。
```

# 9. 分享给别人时，可以怎么理解这两份 Skill？

你可以直接这样介绍：

## 简化介绍版本
这是一套用来做**产品方向判断**的双 Skill 系统。

- 第一个 Skill 负责：**把想法收敛成清晰定位**
- 第二个 Skill 负责：**把方向扩展成商业计划书**
- 每个 Skill 都不是直接写答案，而是会经过：
  - 方法论判断
  - 多智能体讨论
  - 文档生成
  - 自我评估
- 低于阈值会自动回炉，不会直接交一版不成熟的结果

---

# 10. 最推荐的使用姿势

## 场景 A：你只有一个想法
直接这样用：
1. 先跑 `lenny-product-positioning`
2. 通过 evaluation
3. 再跑 `lenny-business-plan`

## 场景 B：你有很多想法
直接这样用：
1. 先让两个 Skill 跑 batch screening
2. 选出 Top 3 / Top 5
3. 再对第一名或前两名做完整深挖

## 场景 C：你已经有清晰定位
那可以直接用：
- `lenny-business-plan`

---

# 11. 常见误区

## 误区 1：一上来就让它写商业计划书
如果定位都没清楚，这通常会写得很空。

## 误区 2：把它当 PRD 生成器
这不是它的职责。

## 误区 3：一次给十几个方向，还要求每个都写完整长文档
这会让输出很散，也不利于比较。

## 误区 4：忽略 evaluation
这套 Skill 的一个关键价值就是：
> **不过线就回炉。**

---

# 12. 一句话总结

> **先用 `lenny-product-positioning` 想清楚“做什么、给谁做、为什么成立”，再用 `lenny-business-plan` 想清楚“怎么成为一门生意”；如果分数不过线，就继续回炉，直到拿到通过版。**
