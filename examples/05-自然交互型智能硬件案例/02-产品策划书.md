# 产品策划书

## 1. 执行摘要

本产品的核心设想是：打造一款**比 OpenClaw 更自然的交互和使用体验**的智能硬件产品，不以“会调用很多工具”为核心卖点，而以“像一个始终在场、低打扰、可感知上下文、能主动协助的人机伙伴”为核心体验。

从产品机会看，OpenClaw 代表的是较强的任务执行与多渠道代理能力，但其主要载体仍偏向聊天式入口、命令式触发和软件工作流。对大量非技术用户、家庭场景用户、轻办公用户而言，真正的障碍不是“AI 不够强”，而是“唤起成本高、交互不够自然、持续陪伴感弱、硬件形态缺位”。

因此，本产品不直接与 OpenClaw 在“通用 agent 工具能力”上正面竞争，而是选择一条更容易形成体验差异的商业路径：**以桌面/床头/办公桌常驻智能硬件为第一载体，聚焦自然交互、环境感知、长时陪伴、轻任务闭环，先拿下高频日常助手场景，再逐步扩展到家庭与团队协同入口。**

本策划书的结论是：
- **主商业路径**：面向高频个人工作与生活场景，推出“自然交互型智能硬件助手”，先以硬件一次性销售 + 会员订阅切入。
- **备选路径**：先从企业/团队前台、会议室、创新部门等半封闭场景切入，做场景化硬件助手方案。
- **关键判断**：比 OpenClaw 更自然，不应理解为“功能更多”，而应理解为“更少命令感、更低使用门槛、更强环境感知、更高陪伴连续性、更接近真实人的协作方式”。

## 2. 市场痛点与机会

### 2.1 当前痛点

当前多数 AI 助手产品存在以下共性问题：

1. **入口仍然过于显性**
   - 大多数产品仍要求用户“打开应用、输入指令、发出命令、等待响应”。
   - 这对技术用户尚可接受，但对普通用户而言，持续使用的心理门槛仍然较高。

2. **交互像工具，不像伙伴**
   - 用户与 AI 的关系更像“临时调用”，而不是“持续协作”。
   - 缺乏环境记忆、弱化了陪伴感，也难以形成习惯依赖。

3. **软件入口天然碎片化**
   - 手机、电脑、群聊、网页、工作台之间切换频繁。
   - 用户知道 AI 能帮忙，但不知道“什么时候叫它最合适”。

4. **主动性和自然性不足**
   - 很多 agent 强在执行，却弱在恰到好处地出现。
   - 不是“叫不动”，而是“不会自然地参与”。

5. **家庭与轻办公场景尚无强势形态**
   - 智能音箱偏弱智化，手机助手偏任务化，机器人偏噱头化。
   - 市场还缺少一个真正把 AI 能力、自然交互和硬件形态统一起来的产品。

### 2.2 市场机会

这类产品的机会不在于教育市场“AI 很厉害”，而在于抓住一个更具体的购买张力：

- 用户希望 AI 不只是回答问题，而是**更自然地帮忙**。
- 用户希望设备**始终在场**，而不是必须打开某个应用才能获得帮助。
- 用户希望它能理解环境、记住偏好、主动提醒、承担轻事务，而不是每次都从零开始。
- 企业与家庭都在寻找新的 AI 入口，但目前还没有成熟的“硬件化自然交互助手”形成标准答案。

### 2.3 为什么是现在

1. 大模型语音理解、实时对话、多模态感知已进入可产品化阶段。
2. 用户对纯聊天机器人已产生审美疲劳，开始追求真实使用体验。
3. 智能硬件市场长期缺少新的高频 AI 原生品类，存在窗口期。
4. OpenClaw 这类 agent 产品证明了任务执行价值，但也暴露出“自然入口”仍未被解决。

## 3. 产品方案概述

### 3.1 产品定义

一款面向个人高频工作与生活场景的**自然交互型智能硬件助手**，核心能力包括：

- 语音优先、可视化辅助的自然对话
- 常驻环境、低唤醒成本
- 面向上下文的连续记忆
- 轻任务闭环执行
- 主动提醒与陪伴式协助
- 与手机/电脑/家庭设备协同

### 3.2 典型硬件形态建议

第一代产品建议采用：

- **桌面式智能终端**：适合办公桌、床头柜、书房、客厅边桌
- 具备麦克风阵列、扬声器、小屏幕/状态灯、摄像头（可物理遮挡）、本地唤醒模块
- 可与手机 App、桌面端、云端账号体系联动

### 3.3 核心产品原则

1. **少命令，多意图理解**
2. **少打开应用，多常驻可达**
3. **少机械响应，多情境化反馈**
4. **少一次性交互，多连续协作**
5. **少炫技，多真实帮助**

### 3.4 与 OpenClaw 的关键差异点

#### 差异点 1：交互入口更自然
- OpenClaw 更像一个强执行的软件代理系统，入口偏聊天、命令和工作流。
- 本产品强调“设备就在你旁边”，可以随时说、随时看、随时打断、随时继续。
- 核心提升是**降低唤起成本**。

#### 差异点 2：环境感知更强
- OpenClaw 的上下文多来自消息、文件、工具、会话。
- 本产品增加物理空间中的时间、位置、声音、在场状态、设备状态等感知。
- 核心提升是**把上下文从数字世界扩展到现实世界**。

#### 差异点 3：陪伴连续性更强
- OpenClaw 强在“收到任务后处理”。
- 本产品强调“一整天都在”，能承接晨间提醒、日间协助、晚间复盘等连续过程。
- 核心提升是**从任务助手走向陪伴式协作伙伴**。

#### 差异点 4：非技术用户门槛更低
- OpenClaw 更适合对 agent、工作流、消息驱动有一定理解的人。
- 本产品强调“像跟一个懂你的设备说话”，更适合家庭成员、管理者、孩子、老人、轻办公人群。
- 核心提升是**扩大可用人群**。

#### 差异点 5：硬件即信任与习惯入口
- 软件 agent 容易被遗忘，使用习惯不稳定。
- 硬件一旦进入固定空间，就天然成为家庭和桌面的注意力入口。
- 核心提升是**提高留存与复访频率**。

#### 差异点 6：主动协助更容易成立
- 在纯软件场景下，主动出现容易打扰。
- 在硬件场景下，可用灯光、屏幕状态、短提示音、环境节奏来做更温和的主动反馈。
- 核心提升是**让“主动”不那么突兀**。

## 4. 目标市场与客户画像

### 用户
- 高强度知识工作者
- 创业者/管理者
- 对 AI 有兴趣但不愿意折腾复杂工具的普通用户
- 家庭场景中的主决策者

### 买方
- 个人消费者
- 家庭主采购人
- 企业创新部门、研究院、前台接待、会议室管理部门

### 决策者
- C 端（Consumer，消费者）主要是本人或家庭主采购人
- B 端（Business，企业客户）主要是部门负责人、行政负责人、信息化负责人、创新负责人

### 第一批切入客户
首批更适合切入的不是大众家庭，而是以下高频高意愿人群：

1. 每天高频使用 AI 的知识工作者
2. 想提升工作流但不想学习复杂软件的人
3. 有桌面固定工作位的管理者和研究人员
4. 对“常驻 AI 助手”有新鲜感和付费意愿的早期尝鲜用户

## 5. 价值逻辑

### 客户买到的结果

客户真正买的不是一台硬件，而是以下结果：

1. **更自然地获得 AI 帮助**
   - 不用频繁打开应用或组织指令。

2. **更低成本地维持高频使用**
   - AI 使用从“偶尔调用”变成“随手协作”。

3. **更持续的提醒、整理与陪伴**
   - 能帮助用户承接日程、想法、待办、信息摘要等碎片工作。

4. **更统一的个人 AI 入口**
   - 将手机、电脑、家庭环境中的 AI 交互收束到一个稳定载体中。

### 为什么愿意切换

用户愿意从手机助手、聊天机器人、智能音箱切换过来，前提是本产品能做到：

- 更自然：像说话，不像下命令
- 更常在：不依赖打开 App
- 更理解场景：知道现在是开会、工作、休息还是准备出门
- 更真实有用：能处理连续的小事，而不是只会回答问题

### 为什么愿意付费

用户愿意付费的逻辑不是“AI 很先进”，而是：

- 节省每天频繁调用 AI 的时间成本
- 提升信息处理和提醒协助的稳定性
- 获得一个更容易形成习惯的 AI 使用入口
- 在情绪和陪伴层面获得额外价值

## 6. 商业模式与定价逻辑

### 收费对象

- C 端个人用户
- 小团队/创新部门/研究院等企业客户

### 收费单位

#### C 端
- 硬件一次性销售
- AI 服务会员按月/年订阅
- 高级能力包增购（如更长记忆、更强模型、家庭协同、专业办公能力）

#### B 端
- 硬件按台采购
- 软件服务按设备数或场景数订阅
- 场景化部署与集成服务收费

### 价格结构逻辑

建议采用“**硬件门槛可接受 + 会员体现持续价值**”的结构。

#### C 端建议
- 硬件售价：中高端智能音箱/桌面设备区间
- 基础能力：随硬件附带
- 进阶能力：通过会员解锁

#### B 端建议
- 硬件作为部署入口
- 真正收入重点放在持续订阅与场景服务

### 定价逻辑背后的原因

1. 纯卖硬件很难建立长期收入与产品迭代能力。
2. 纯订阅又难以解释“为什么不直接用软件助手”。
3. 硬件 + 会员更符合“入口 + 持续服务”的价值结构。

### 毛利风险

- 语音实时交互带来持续推理成本
- 多模态感知和主动提醒会提高云端成本与支持成本
- 若订阅率不足，硬件利润难以覆盖长期服务投入

因此必须在早期控制：
- 高频功能的模型成本
- 主动交互的触发频率
- 高级能力与基础能力的边界

## 7. GTM（Go-To-Market，市场进入）策略

### 第一批客户

第一批不宜追求大而全，应从“高频、强痛点、可复购传播”的用户群切入：

1. AI 重度用户
2. 知识工作者
3. 创新管理者
4. 智能硬件早期用户

### 信任机制

这个品类最大的购买障碍不是“有没有兴趣”，而是“是否可信、是否真有用”。

必须重点建立：
- 隐私可信：摄像头物理遮挡、状态灯提示、本地唤醒、权限可见
- 使用可信：真实场景演示，而不是概念宣传片
- 品牌可信：清晰说明它能做什么、不能做什么

### 激活点

产品的第一个价值时刻不应是“它回答了一个问题”，而应是：

- 它在一天内连续帮用户完成 3—5 个轻任务
- 它在用户未打开任何应用的前提下完成提醒、记录、查询、整理、转发中的若干动作
- 用户感知到“它真的更自然、更顺手”

### 扩张路径

#### C 端扩张
- 从单人桌面扩展到家庭多场景
- 从语音助手扩展到家庭协作中心
- 从提醒与问答扩展到设备联动、家庭知识记忆和家庭事务管理

#### B 端扩张
- 从单点试用扩展到会议室、前台、创新空间
- 从个人助手扩展到团队知识入口、会议助手、运营协助终端

### 最弱 GTM 假设

最脆弱的假设是：**用户是否真的愿意为“更自然”单独购买一个新硬件入口。**

因此初期必须用足够清晰的高频场景证明：
- 它不是智能音箱换皮
- 也不是把聊天机器人塞进硬件壳子里
- 而是明显改善了 AI 使用体验

## 8. 竞争与替代分析

### 主要替代品

1. OpenClaw 等软件型 agent
2. ChatGPT、Claude 等通用聊天式助手
3. 手机语音助手
4. 智能音箱
5. 屏幕音箱、桌面机器人、陪伴型硬件

### 竞争判断

#### 相比 OpenClaw
- OpenClaw 更强在代理执行、消息通路、工作流编排
- 本产品更强在自然入口、陪伴连续性、物理空间常驻和低唤起成本

#### 相比 ChatGPT / Claude
- 通用模型更强在知识与生成
- 本产品如果不能把硬件场景做深，就会沦为昂贵入口

#### 相比智能音箱
- 智能音箱已验证了硬件入口的需求
- 但其智能程度和任务能力偏弱，本产品有升级空间

### 为什么还没有被大厂彻底拿下

1. 这个品类需要同时把硬件、语音、模型、隐私、主动交互、习惯设计做好，难度高。
2. 过去模型不够强，做不出真正自然的体验。
3. 大厂已有产品包袱，容易停留在“助手升级版”，而不是重新定义体验。

### 防御性来源

本产品真正可能形成的防御性，不在于“我们也有 AI”，而在于：

- 高频真实场景下的体验打磨
- 环境上下文与个体记忆积累
- 设备在固定物理空间中的入口地位
- 用户习惯形成后的切换成本
- 隐私与可信交互机制带来的品牌信任

## 9. 风险与应对

### Top 3 风险

#### 风险 1：价值不够强，用户不愿单独购买硬件
**应对：**
- 先聚焦高频重度用户
- 用连续轻任务闭环证明价值
- 控制第一代产品功能范围，避免泛化

#### 风险 2：产品看起来新鲜，但长期留存不足
**应对：**
- 重点设计晨间、工作中、晚间三个高频使用链路
- 强化记忆、提醒、连续协作能力
- 用周度/月度复盘提升习惯黏性

#### 风险 3：隐私和主动交互处理不好，造成反感
**应对：**
- 坚持显性权限、可关闭主动提醒、摄像头物理控制
- 明确“主动但不过度打扰”的产品边界
- 把信任设计视为核心功能，不视为合规附属项

### 最脆弱假设

最脆弱假设不是技术能不能实现，而是：**用户是否会因为“更自然”而改变现有使用习惯，并为一个新入口买单。**

## 10. 里程碑规划

### 第一阶段：方向验证（0—2个月）
目标：证明“更自然”不是抽象概念，而是可被用户感知的强体验差异。

重点动作：
- 完成产品概念定义与核心场景拆解
- 明确与 OpenClaw 的体验差异模型
- 做低保真原型与交互脚本验证
- 访谈 15—20 位目标用户
- 选出 3 个最高频核心场景

阶段输出：
- 产品定位说明
- 场景清单
- 差异点验证报告

### 第二阶段：最小可行产品（Minimum Viable Product，最小可行产品）验证（2—5个月）
目标：做出能体现“自然交互优势”的最小闭环。

重点动作：
- 开发第一版桌面硬件原型
- 打通语音唤醒、连续对话、记忆、提醒、轻任务执行
- 完成 20—50 名种子用户测试
- 记录留存、日活、使用时长、复访场景

阶段输出：
- MVP 原型
- 用户测试报告
- 功能取舍清单

### 第三阶段：首批商业化试点（5—9个月）
目标：验证用户愿意付费和愿意持续使用。

重点动作：
- 上线首批预售/邀请制体验计划
- 验证硬件售价带和会员接受度
- 形成 C 端种子口碑与 B 端试点案例

阶段输出：
- 首批付费用户数据
- 定价修正方案
- 商业化试点复盘

### 第四阶段：规模化准备（9—15个月）
目标：从概念产品过渡为可持续经营的产品线。

重点动作：
- 优化供应链和硬件成本
- 完善 App 与云端协同体系
- 形成家庭版/办公版场景扩展方案
- 明确渠道和品牌传播打法

阶段输出：
- 量产准备方案
- 渠道测试方案
- 第二代产品路线图

## 11. 当前结论

### 主商业路径

以**个人桌面/床头常驻智能硬件助手**为切入点，突出“比 OpenClaw 更自然”的体验优势，即：
- 更低唤起成本
- 更强现实环境感知
- 更连续的陪伴协作
- 更适合非技术用户
- 更适合作为个人 AI 入口硬件

采用“硬件销售 + 会员订阅”的商业模式，先服务高频 AI 用户和知识工作者，再向家庭和团队扩展。

### 备选路径

若 C 端单独买硬件的教育成本过高，可优先切入企业/研究院/会议室/前台等半封闭场景，作为“场景化 AI 助手终端”进行试点，再回到消费市场。

### 不应过度宣称的点

- 不应把“自然交互”宣传成拟人化奇迹
- 不应假设用户会因为喜欢 AI 就愿意买新硬件
- 不应把语音、多模态、本地感知本身当作壁垒
- 不应把 OpenClaw 简单定义为“差”，而应定义为“侧重点不同，本产品解决的是它未优先解决的问题”
