# 案例说明：自然交互型智能硬件完整案例

这个案例展示了一次完整的双技能执行链路：

1. 先使用 `lenny-product-positioning`
2. 再使用 `lenny-product-plan`

案例目标是说明：当用户只有一个相对抽象的产品想法——“做一个比 OpenClaw 更自然的交互和使用体验的智能硬件产品”——如何先把定位收敛清楚，再进一步形成结构化产品策划书。

## 原始提示词

```text
用产品规划和策划设计skill，针对一下想法形成对应的产出。想法：比open claw更自然的交互和使用体验的智能硬件产品规划 分析出比openclaw更自然的差异点和落地计划

定位研讨也形成md，两个skill都要执行
```

## 生成结果

- `01-产品定位研讨.md`
- `02-产品策划书.md`

## 使用意义

这个案例适合说明：
- 面对一个偏抽象、偏愿景型的 AI 硬件方向，如何先做定位，再做产品策划书
- 如何把“比某个现有产品更自然”拆成可表达的结构性差异，而不是停留在感性描述
- 两个 skill 串联后，如何输出稳定成品，而不是聊天草稿
